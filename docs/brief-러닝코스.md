# 기능 브리프: 러닝코스 추천 (M4)

> 런닝구(RunTrip)의 M4 기능을 PRD로 발전시키기 위한 브리프. 근거: 이 세션의 API 실측(docs/SPEC.md §6.4), 경쟁 조사(research-competitors.md), 선경과의 코칭 대화(.memlog.md). 민지의 목업은 푸시 대기 중 — 화면 상세는 PRD 단계에서 반영.

## 한눈에

처음 간 여행지에서 "이 근처에서 5km 뛸 만한 곳"을 알려주는 서비스가 한국에 없다. 두루누비 공식 앱조차 수 시간짜리 걷기여행 단위라 러닝 거리로 찾을 수 없고, 가장 근접했던 스트라바는 2025년 3월 한국에서 철수했다. 런닝구 M4는 한국관광공사 두루누비 걷기길 데이터를 **러닝 거리 단위로 재가공**해, 원정 러너가 내 주변 기준으로 원하는 거리의 코스를 30초 안에 찾게 한다. 공모전 필수요건(KTO OpenAPI 활용)과 정확히 겹치는, 비어 있는 자리다.

## 문제

**주인공: 원정 러너.** 대회를 뛰러 처음 가는 지역에 온 러너는 "내일 아침 몸 풀 코스"를 찾을 방법이 없다. 지도앱에서 '공원'을 검색하거나 블로그를 뒤지거나 숙소 주변을 무작정 달리는데, 거리도 난이도도 확신할 수 없다.

> "처음 간 여행지에서 주변에 뛸만한 코스를 알 수가 없어서 검색한다." — 문제 문장 (선경, 7/2)

**부차 수혜자: 일상 러너.** 같은 기능이 여행 없이도 동작하므로 동네 코스 탐색에도 쓸 수 있다. 단 이 시장엔 신생 앱(런세권, Runnect)이 진입 중이므로 1순위가 아니다 — 화면과 성공 기준은 원정 시나리오에 맞춘다.

## 해법

**"내 주변 + 원하는 거리 → 달릴만한 코스"**

- **내 주변**: GPS 우선, 권한 거부·실패 시 여행지(선택한 대회장·숙소) 기준 폴백. GPS는 HTTPS 배포에서만 동작.
- **원하는 거리**: 거리 필터(예: 3~5km / 5~10km)로 설정.
- **달릴만한**: 원하는 거리 ± 난이도 3단계. 두루누비 crsLevel(하/중/상)을 러너 언어로 번역한다 — "초보 OK / 무난 / 도전".

**구조적 전제** (실측 제약): 두루누비 API는 위치 검색이 안 된다. 따라서 전량 수집 → GPX 파일에서 시작점 좌표 추출·색인 → 사용자 위치와 로컬 거리 계산 방식이다. 보조 소스로 국문관광 #01 여행코스(반경 검색 가능)와 카카오 키워드(둘레길·산책로)를 쓴다.

**모르는 것 (정직하게)**: 조명·안전·노면 데이터는 두루누비에 없다 — 지어내지 않고 표시하지 않는다(코스 설명 텍스트에 있으면 그대로 노출). 경사는 GPX 고도에서 계산 가능하나 MVP 이후로 미룬다.

## 차별점

1. **거리 기준 검색의 국내 부재** — "이 근처에서 N km"에 답하는 서비스가 없다 (경쟁 조사, 7/2).
2. **타이밍** — 최근접 경쟁자 스트라바가 2025.3 한국 철수 (그마저 유료 기능이었다).
3. **여행·대회 맥락 결합** — 코스 앱들은 숙소·대회와 연결이 없고, 러닝 앱들은 코스 개념이 없다. "대회 전날 숙소 근처에서 가볍게"를 조합하는 것은 우리뿐.
4. 해자는 데이터 보유가 아니라(두루누비는 공개 데이터) **재가공(러닝 거리 단위) + 맥락(대회·여행·회복 룰)**이다.

## 성공 기준

| # | 기준 | 측정 |
|---|---|---|
| 1 | 커버리지 — 두루누비를 "잘" 활용 | 걷기길 전량 수집·색인 완료. 271개 대회 중 반경 20km 내 코스 3개 이상 매칭 비율 — **수집 후 실측하여 목표치 확정** |
| 2 | 적합성 — "적합한" 코스 추천 | 설정 거리 ±2km 코스가 목록 상위에 노출 |
| 3 | 사용성 | 코스 탭 진입 → 코스 선택까지 3탭·30초 이내 |
| 4 | 심사 증빙 | 코스 화면에 "출처: 한국관광공사 두루누비" 상시 표기 |

## 범위

**MVP (8/14 앱 완성에 포함)**
- 내 주변 기준 코스 목록 + 거리·난이도 필터 + 코스 상세(길이·난이도·소요시간·설명·시작점)
- 위치: GPS → 여행지 폴백 / 출처 표기 / 기존 스텁 UI(CoursesScreen)에 데이터 연결

**MVP 이후 (제출 전 여유 시, 이 순서로)**
1. 지도에 실제 경로 표시 (GPX 폴리라인)
2. 여행 동선에 코스 담기 (M1 walk 블록 연계 — 우리만의 연결 고리)
3. 코스 상세에서 "시작점으로 안내" (카카오맵/내비 연결)
4. 코스 저장·카톡 공유

**명시적 제외**
- 조명·안전 지표 (데이터 없음), 경사 표시 (GPX 고도 — 로드맵), 코스 그리기·사용자 생성 코스, 비러너 확장(보류 — addendum 참조)

## 비전

코스에 러너 관점 데이터(경사, 완주 기록, 한줄평)가 쌓이면 "걷기길 DB의 러닝 번역판"이라는 독자 자산이 된다. 대회와 독립적으로 동작하는 이 탭은 비대회 시즌·비러너 확장의 씨앗이기도 하다 (확장 여부는 팀 논의 — addendum).

## 미결·리스크

1. **민지 목업 미반영** — 깃헙 푸시 후 화면 상세를 PRD에서 정합.
2. **걷기길의 러닝 적합성 미실측** — 두루누비 코스 길이 분포가 대부분 10km 이상이면 거리 필터 설계(구간 분할 여부)에 영향. 전량 수집 직후 분포 확인 필요.
3. **커버리지 미실측** — 성공 기준 1의 목표치는 수집 후 확정.
4. GPX 파싱·색인은 새로 만드는 코드 — 수직 슬라이스(경주 1건)에 코스 1~2개를 포함해 조기 검증 권장.
