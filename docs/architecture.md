# 초기 구조 설계

## 추천 개발 환경

우선 추천은 Java 17 + Spring Boot 3.x입니다.

- Java 17: 현재 PC에도 JDK 17이 설치되어 있어 시작 비용이 낮습니다.
- Spring Boot: 공공데이터 REST API 연동, 배치, 캐시, DB, 인증키 관리에 강합니다.
- Gradle: 의존성 관리와 CI 구성이 단순합니다.
- PostgreSQL/PostGIS: 위치 기반 관광 데이터를 다룰 가능성이 높으므로 지리 검색까지 확장하기 좋습니다.
- Redis: 공공 API 호출량 제한과 응답 속도 개선을 위한 캐시로 적합합니다.
- React 또는 Next.js: 지도/검색/필터 UI를 빠르게 만들기 좋습니다.

## 백엔드 모듈 초안

| 모듈 | 역할 |
| --- | --- |
| `tour-api-client` | VisitKorea/data.go.kr API 호출, 공통 파라미터, 인증키, 에러 처리 |
| `catalog` | 관광지/사진/반려동물/무장애 등 API별 도메인 모델 정리 |
| `search` | 지역, 키워드, 위치 기반 검색 및 정렬 |
| `recommendation` | 사용자 조건 기반 코스/장소 추천 로직 |
| `cache` | API 응답 캐싱, 일별/시간별 갱신 정책 |
| `contest` | 공모전 제출용 설명 자료, 성과 지표, 데모 데이터 관리 |

## 데이터 흐름

1. 사용자가 지역, 동행 조건, 목적을 입력합니다.
2. 백엔드는 선택 조건을 VisitKorea API 요청 파라미터로 변환합니다.
3. 외부 API 응답은 표준 내부 DTO로 정규화합니다.
4. 주요 응답은 Redis 또는 DB에 캐시합니다.
5. 서비스 도메인 로직이 사용자에게 코스, 장소, 사진, 보조 정보를 제공합니다.

## 우선 검토 API 조합

| 서비스 방향 | 핵심 API | 보조 API |
| --- | --- | --- |
| 반려동물 동반 여행 | 반려동물 동반여행 서비스 | 국문 관광정보, 관광사진 정보 |
| 무장애 여행 추천 | 무장애 여행 정보 | 국문 관광정보, 관광사진 정보 |
| 혼잡 회피 여행 | 관광지 집중률 방문자 추이 예측 정보 | 관광빅데이터 정보서비스 |
| 감성 사진 여행 | 관광사진 정보 | 관광공모전 사진 수상작 정보, 국문 관광정보 |
| 지역 관광 수요 분석 | 지역별 관광 다양성/수요 강도/자원 수요 | 관광빅데이터 정보서비스 |

## 초기 구현 순서

1. Spring Boot 프로젝트 생성
2. 공통 API 클라이언트 구현: `serviceKey`, `MobileOS`, `MobileApp`, `_type=json`
3. 국문 관광정보 서비스 1개 엔드포인트 연동
4. 캐시와 로깅 추가
5. 서비스 아이디어에 맞는 API 2~3개 추가
6. 제안서용 아키텍처 다이어그램과 사용자 시나리오 작성
