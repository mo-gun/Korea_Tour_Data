# 2026 관광데이터 활용 공모전 공고 요약

## 원본 파일

- 공고문: [../../assets/contest/2026-tour-data-contest-notice.pdf](../../assets/contest/2026-tour-data-contest-notice.pdf)
- 제안서 양식: [../../assets/contest/2026-tour-data-contest-proposal-template.hwp](../../assets/contest/2026-tour-data-contest-proposal-template.hwp)

## 핵심 내용

- 공모명: 2026 관광데이터 활용 공모전, 웹/앱 개발 부문
- 주최/협력: 한국관광공사, 카카오 공동 개최로 공개된 공모
- 접수기간: 2026.03.30 09:00 ~ 2026.05.06 16:00
- 접수방법: 한국관광 콘텐츠랩 공모전 홈페이지 온라인 접수
- 참가자격: 대한민국 국민 누구나, 국내 거주자 한정, 개인 또는 팀 참가 가능
- 팀 구성: 팀당 최대 5명
- 공모대상: 한국관광공사 OpenAPI를 필수 활용한 신규 웹/앱 관광 서비스
- 선발규모: 총 31개 팀
- 주요 일정: 예비심사 및 OT 5월, 서비스 개발 및 컨설팅 5월~9월, 기능 심사 10월, 최종 심사 10월, 시상식 11월

## 시상

| 구분 | 규모 | 상금 |
| --- | ---: | ---: |
| 대상 | 1팀 | 1,000만원 |
| 최우수상 | 5팀 | 각 300만원 |
| 우수상 | 10팀 | 각 100만원 |
| 장려상 | 15팀 | 각 50만원 |
| 특별상 | 기관별 1팀 | 별도 |

대상은 웹/앱 구현 부문 공모전과 통합 심사 후 1팀 선정 예정으로 공개되어 있습니다. 심사 결과에 따라 선정 규모는 바뀔 수 있습니다.

## 준비 관점

- 제안서에는 서비스 문제 정의, 활용 API, 사용자 흐름, 차별성, 구현 계획, 데이터 처리 구조를 명확히 써야 합니다.
- 공모전 성격상 단순 API 조회보다 실제 사용자가 반복적으로 쓸 수 있는 서비스 가치가 중요합니다.
- 한국관광공사 OpenAPI 활용은 필수 조건이므로, API 카탈로그에서 최소 1개 핵심 API와 보조 API를 미리 선정해 두는 편이 좋습니다.
- 카카오와 연계된 테스트베드/사업화 가능성을 고려하면 지도, 위치, 채널, 공유, 알림 등 사용자 접점이 있는 구조가 유리합니다.
