## 1. 우리가 만드는 것

**런닝구(區)** — 전국 마라톤 대회 일정을 한곳에 모으고, 대회를 고르면 대회 전날~다음날 여행 일정을 지도 위에 자동으로 짜 주는 서비스.

**왜 필요한가**: 마라톤 일정은 블로그·인스타·개별 홈페이지에 흩어져 있어서 믿을 만한 통합 서비스가 없다. 그리고 "하프 뛰고 나서 다음날 뭐 하지?"에 답해주는 서비스도 없다.

**우리만의 차별점 4개** (제안서에서 약속한 것):

| # | 차별점 | 쉽게 말하면 | 현재 상태 |
|---|---|---|---|
| D1 | 대회→여행 원스톱 | 대회 고르면 식당·관광지·축제까지 지도에 한 번에 | 앱에 구현됨 |
| D2 | 자유 편집 | 추천받은 일정에서 장소를 빼고·바꾸고·순서 조정 | 구현됨 (폰 드래그만 보완 필요) |
| D3 | 러너 컨디션 반영 | 풀코스 뛴 다음날은 등산 대신 온천을 추천 | 구현됨 (핵심 자랑거리) |
| D4 | 비시즌 커버 | 대회 없어도 "오늘 뛸 코스" 추천 | 화면만 있음, 데이터 연결 필요 |

## 2. 앱 화면과 흐름

사용자가 겪는 순서 그대로:

```
① 홈(대회 목록) → ② 대회 상세 → ③ 언제 갈지 → ④ 종목·취향 → ⑤ 숙소 → ⑥ 완성된 동선
```

| 화면 | 하는 일 |
|---|---|
| ① 홈 | 대회 검색·필터(지역/월/접수중). 카드마다 "출처·확인일"을 보여줘서 믿을 수 있게 |
| ② 대회 상세 | D-day, 종목, 접수 기간, 공식 홈페이지 링크, 근처 축제 |
| ③ 일정 | 4가지 패턴 중 선택: 전날부터 / 대회+다음날 / 전후로(2박3일) / 당일치기 |
| ④ 취향 | 뛸 종목(5K/10K/하프/풀) + 여행 취향(맛집, 관광지, 카페, 힐링, 자연, 역사) |
| ⑤ 숙소 | 대회장 근처 숙소 고르기 (건너뛰기 가능) |
| ⑥ 결과 | 지도 + 시간표로 일정 표시. 편집·저장·공유 |
| +러닝코스 탭 | 대회 없는 날 뛸 코스 (D4) |
| +내 여행 탭 | 저장한 일정 보관함 |

## 3. 추천 규칙 — 앱의 두뇌

핵심 아이디어: **"어떤 종목을 뛰느냐에 따라 몸 상태가 다르다."**

| 종목 | 완주 후 몸 상태 | 앱이 하는 일 |
|---|---|---|
| 5K / 10K | 거의 정상 | 완주 후 오후부터 관광·카페·맛집 일정 |
| 하프 | 중간 피로 | 완주 후 온천·휴식, 다음날은 짧은 산책만 |
| 풀 | 회복 필요 | 관광 최소화, 스파·온천 중심, 걷기 3km 이하 |

하루 일정은 이렇게 짜입니다:

- **대회 전날**: 오후 숙소 체크인 → 저녁은 탄수화물(카보로딩) → 가벼운 산책
- **대회 당일**: 스타트 → (하프·풀이면) 온천·회복 / (5K·10K면) 관광·카페 → 저녁 → 산책
- **다음날**: 아침 산책 → (하프·풀이면) 온천·족욕 / 아니면 관광 → 점심 → 관광 → 귀가

같은 장소가 일정에 두 번 나오지 않게 하고, 사용자가 고른 취향(맛집·역사 등)을 우선 배치합니다.

## 4. 데이터가 흐르는 길

**서버가 없습니다.** 미리 만들어 둔 파일을 앱이 읽는 방식입니다. (돈 안 들고, 시연 때 안정적)

```
[미리 준비 — 우리 컴퓨터에서 실행]
대회 271개 CSV ──변환──▶ races.json (대회 목록 파일)
관광공사 API ──대회마다 주변 장소 수집──▶ pois/대회이름.json (장소 파일)

[앱 실행 중]
앱이 위 파일들을 읽음 + 카카오 지도로 표시
인터넷·키가 안 되면? → 미리 저장한 샘플 → 그래도 없으면 합성 데이터 (앱이 절대 안 죽게 3단 안전망)
```

- 대회 데이터: 271개 (마라톤온라인 138 + 마라톤GO 133), 전부 좌표 확보.
- 알려진 데이터 문제 2개: ① 지역명에 '충청' 같은 비표준 값 → 정제 필요 ② 종목 정보가 없는 대회 45개(트레일·걷기 등) → 목록엔 보여주되 추천은 5K 기준으로.

## 5. 우리가 쓰는 외부 서비스 (전부 7/2 실제 호출로 검증 완료)

| 서비스 | 우리 앱에서 쓰는 곳 | 상태 |
|---|---|---|
| 관광공사 **국문관광정보** | 대회 주변 관광지·음식점 / 축제(M3) | ✅ 검증 완료 |
| 관광공사 **두루누비** | 걷기 코스 (M4, 러닝코스 탭) | ✅ 검증 완료 (단, 좌표를 따로 뽑아야 함) |
| 관광공사 **웰니스** | 온천·스파 (하프·풀 회복 일정의 핵심) | ✅ 검증 완료 (7/2 신청 완료) |
| 카카오 **지도** | 앱 안의 지도 | ✅ 키 유효, 도메인 등록 완료 |
| 카카오 **로컬 검색** | 맛집·카페·숙소 찾기, 주소→좌표 변환 | ✅ 검증 완료 |
| 카카오 **길찾기(모빌리티)** | 장소 사이 이동 시간 표시 | ✅ 검증 완료 (기존 키로 됨!) |
| 카카오 공유·내비 | 카톡 공유, "내비로 출발" | 문서 확인만 (구현 시 검증) |

**관광공사 API 사용은 공모전 필수 요건**입니다. 관광지·축제·온천·코스가 관광공사 데이터로 나오는 순간 충족됩니다.

## 6. 조심할 것 (개발할 때 밟기 쉬운 지뢰 8개)

1. **좌표 순서 함정**: 카카오 검색은 (경도, 위도) 순서, 카카오 지도는 (위도, 경도) 순서. 반대다! 헷갈리면 마커가 바다에 찍힘.
2. **옛날 지역코드 함정**: 축제 검색에 옛 지역코드(areaCode)를 쓰면 **에러도 없이 결과만 0건**. 새 코드(lDongRegnCd)만 쓸 것.
3. **구버전 API 금지**: 블로그의 KorService1 예제 코드 복붙하면 무조건 실패(500 에러). 우리는 KorService2.
4. **하루 1,000번 제한**: 관광공사 API는 기능당 하루 1,000번까지. 271개 대회 × 여러 카테고리를 한 번에 돌리면 초과 → 이틀에 나눠 실행.
5. **에러가 JSON이 아님**: 관광공사가 한도 초과 등일 때 JSON을 달라고 해도 XML로 답함 → 파싱 전에 확인.
6. **CSV 행 수 ≠ 대회 수**: 설명 칸에 줄바꿈이 있어서 898줄이지만 대회는 271개. 제대로 된 CSV 파서 필수.
7. **키 두 종류**: 관광공사 키는 "인코딩 키"와 "디코딩 키" 페어 — URL에 직접 붙이면 인코딩 키, 라이브러리에 넘기면 디코딩 키. 혼용하면 인증 실패.
8. **배포하면 지도 확인**: 카카오 지도는 등록된 주소에서만 동작. 배포 주소가 정해지면 콘솔에 꼭 추가.

## 7. 남은 일 (우선순위순, SPEC.md 백로그를 쉬운 말로)

**P0 — 이것 없으면 서비스가 아님**
- 가짜 대회 7개 → 진짜 271개로 교체 (races.json 연결)
- 앱 안의 날짜가 6/18로 고정돼 있음 → 실제 오늘 날짜로
- 대회 상세의 축제가 경주 고정 → 진짜 축제 데이터로
- 주변 장소 미리 수집 (지금은 경주 1개 대회만 있음 → 전체 대회로)
- 러닝코스 탭에 두루누비 진짜 데이터 연결

**P1 — 완성도**
- 홈 캘린더 뷰 (지금은 버튼만 있고 동작 안 함)
- 카톡 공유 실제 구현 (지금은 버튼만)
- 이동 시간 표시
- 대회 포스터 이미지 표시 (133개 보유)
- 저장한 일정이 새로고침하면 사라짐 → 브라우저에 저장
- 편집 드래그 폰 대응 (dnd-kit)

**P2 — 여유 있으면**
- 카카오 로그인, 내비 연동, 반려동물 필터, 다국어

## 8. 용어 사전

| 용어 | 뜻 |
|---|---|
| API | 다른 회사 데이터를 코드로 받아오는 창구. "관광공사 API 호출" = 관광공사에 데이터 요청 |
| POI | 지도 위의 장소 하나 (식당, 관광지, 온천 등. Point of Interest) |
| 지오코딩 | 장소 이름·주소 → 좌표(위도/경도)로 바꾸는 것 |
| 크롤링 | 웹사이트에서 정보를 긁어오는 것 (우리: 마라톤 대회 목록) |
| 파이프라인 | 데이터를 수집→정리→변환하는 일련의 자동 과정 |
| JSON | 앱이 읽기 좋은 데이터 파일 형식 (races.json 등) |
| CSV | 엑셀 같은 표 형태의 데이터 파일 (수집 원본) |
| 폴백 | 1순위가 실패하면 2순위로 자동 전환하는 안전망 |
| 쿼터 | API 하루 사용 한도 (관광공사: 기능당 1,000번) |
| SDK | 카카오 지도를 앱에 넣을 때 쓰는 카카오 제공 코드 뭉치 |
| JS 키 / REST 키 | 카카오 키 2종. JS 키 = 화면(지도)용·공개돼도 됨 / REST 키 = 데이터 수집용·**절대 공개 금지** |
| GPX | 걷기 코스의 경로 좌표가 든 파일 (두루누비가 이 형식으로 줌) |
| D-1 / D-day / D+1 | 대회 전날 / 당일 / 다음날 |
| M1~M4 | 제안서의 기능 번호: M1 동선 추천, M2 대회 캘린더, M3 인근 축제, M4 러닝 코스 |
