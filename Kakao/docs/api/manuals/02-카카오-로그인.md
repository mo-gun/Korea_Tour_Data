# 02. 카카오 로그인 (Kakao Login)

> 출처: <https://developers.kakao.com/docs/ko/kakaologin/common>

OAuth 2.0 기반 소셜 로그인. 런트립의 "원터치 회원가입·로그인" 기반.

## 개요

카카오톡/카카오계정으로 별도 회원가입 절차 없이 서비스 로그인. 동의항목(scope)을 통해 프로필·이메일 등 사용자 정보를 받아온다.

## 지원 플랫폼

REST API · JavaScript SDK · Android SDK · iOS SDK · Flutter

## 인증 흐름 (Authorization Code Grant)

```
[1] 클라이언트 → 카카오 인가 코드 요청
[2] 사용자 카카오 인증 + 동의 화면 진행
[3] 카카오 → 서비스 redirect_uri로 인가 코드 전달
[4] 서비스 서버 → 카카오 토큰 엔드포인트로 인가 코드 + 시크릿 전송
[5] 카카오 → access_token / refresh_token / (선택) id_token 응답
[6] 서비스 서버 → access_token으로 사용자 정보 조회
```

## 주요 엔드포인트

| 기능 | 메서드 | URL |
|---|---|---|
| 인가 코드 요청 | GET | `https://kauth.kakao.com/oauth/authorize` |
| 토큰 발급 / 갱신 | POST | `https://kauth.kakao.com/oauth/token` |
| 사용자 정보 조회 | GET/POST | `https://kapi.kakao.com/v2/user/me` |
| 로그아웃 | POST | `https://kapi.kakao.com/v1/user/logout` |
| 연결 끊기 | POST | `https://kapi.kakao.com/v1/user/unlink` |

### 인가 코드 요청 (예시)

```
GET https://kauth.kakao.com/oauth/authorize
  ?client_id={REST_API_KEY}
  &redirect_uri={REDIRECT_URI}
  &response_type=code
  &scope=profile_nickname,profile_image,account_email
  &prompt=login            # 옵션 — 강제 재로그인
```

### 토큰 발급 (예시)

```
POST https://kauth.kakao.com/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&client_id={REST_API_KEY}
&redirect_uri={REDIRECT_URI}
&code={AUTH_CODE}
&client_secret={CLIENT_SECRET}   # 사용 설정 시 필수
```

응답 예시:
```json
{
  "token_type": "bearer",
  "access_token": "...",
  "expires_in": 21599,
  "refresh_token": "...",
  "refresh_token_expires_in": 5183999,
  "scope": "profile_nickname account_email",
  "id_token": "..."
}
```

### 사용자 정보 조회

```
GET https://kapi.kakao.com/v2/user/me
Authorization: Bearer {ACCESS_TOKEN}
```

## 동의항목 (대표)

문서에서 언급된 항목 중 런트립 사용 가능 후보:
- 프로필 정보 (`profile_nickname`, `profile_image`)
- 이메일 (`account_email`)
- 성별 / 연령대 — 러너 페르소나 분석용 (B2B KPI 단계)
- 친구 정보 — 카톡 메시지/공유 활용 시
- 배송지 정보 — 발전 방향(굿즈·물품 보관 패키지) 단계

> 일부 항목은 비즈 앱 전환·심사 후 사용 가능.

## 토큰 유효기간

| 토큰 | 역할 | 만료 |
|---|---|---|
| **Access Token** | API 호출 인증 | Android/iOS 12h, JS 2h, REST 6h |
| **Refresh Token** | Access Token 갱신 | 60일 (만료 30일 전부터 갱신 가능) |
| **ID Token** | OpenID Connect 활성화 시 발급 | Access Token과 동일 |

> 토큰 갱신 시 새 Refresh Token이 발급되면 기존 Refresh Token은 폐기됨.

## 런트립 활용 매핑

| 기획안 기능 | 사용 방식 |
|---|---|
| 원터치 로그인 | 카카오 로그인 — 가장 빠른 가입 경로 |
| 사용자 GPS·선택 행동 누적 (B2B 자산화) | `kakao_account.profile.id`(서비스 내부 user_id 매핑) 기반 |
| 추천 일정 선택률·인근 시설 방문 데이터 | 사용자 식별 후 행동 로그 적재 |
| 동반자 분리형 트랙 | 동행자 초대 시 카톡 친구 정보 활용 (선택) |

## 보안 체크

- `client_secret`는 서버에서만 보관·사용 (코드/리포지토리 노출 금지)
- `redirect_uri`는 카카오디벨로퍼스에 등록된 정확한 값과 1:1 매칭
- 토큰은 HTTP-only Secure 쿠키 또는 서버 세션에 보관
