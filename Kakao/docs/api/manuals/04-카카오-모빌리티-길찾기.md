# 04. 카카오 모빌리티 길찾기 API

> 출처: <https://developers.kakaomobility.com/guide/navi-api/> · <https://developers.kakaomobility.com/guide/navi-api/start.html>

런트립의 **"숙소 ↔ 대회장 ↔ 관광지 실제 이동시간"** 산출 핵심. 카카오 본사 디벨로퍼스가 아닌 **카카오 모빌리티 디벨로퍼스(별도)** 에서 키 발급.

## 호스트

```
https://apis-navi.kakaomobility.com
```

## 제공 API 비교

| API 이름 | 출발지 | 목적지 | 경유지 | 출발 기준 | 제공 정보 |
|---|---|---|---|---|---|
| **자동차 길찾기** | 1개 | 1개 | 최대 5개 | 현재 | 구간별 경로 상세 또는 요약 |
| **다중 경유지 길찾기** | 1개 | 1개 | 최대 30개 | 현재 | 구간별 경로 상세 또는 요약 |
| **다중 출발지 길찾기** | 최대 30개 | 1개 | — | 현재 | 예상 소요 시간, 이동 거리 |
| **다중 목적지 길찾기** | 1개 | 최대 30개 | — | 현재 | 예상 소요 시간, 이동 거리 |
| **미래 운행 정보 길찾기** | 1개 | 1개 | 최대 5개 | **미래** | 구간별 경로 상세 또는 요약 |

## 엔드포인트 목록

| API | 엔드포인트 | 메서드 |
|---|---|---|
| 자동차 길찾기 | `/v1/directions` | GET |
| 다중 경유지 길찾기 | `/v1/waypoints/directions` | POST |
| 다중 출발지 길찾기 | `/v1/origins/directions` | POST |
| 다중 목적지 길찾기 | `/v1/destinations/directions` | POST |
| 미래 운행 정보 길찾기 | `/v1/future/directions` | GET |

## 인증 / 사전 설정

1. 카카오디벨로퍼스(developers.kakao.com) 로그인 — 동일 계정 사용
2. 앱 생성 (앱 아이콘 128×128 ≤ 250KB / 앱 이름 30자 / 회사명 / 카테고리 / 대표 도메인)
3. 플랫폼 키 → REST API 키 추가
4. 서비스 정보 등록 (키 이름, IP 주소 등)
5. REST API 키 확인 및 복사

> 가격/트래픽 한도는 별도 협의: <https://developers.kakaomobility.com/price/>

## 호출 예시 (자동차 길찾기 — 추정)

```bash
curl -X GET "https://apis-navi.kakaomobility.com/v1/directions?\
origin=129.0756416,35.1795543&\
destination=129.1310,35.1535&\
waypoints=129.0900,35.1700&\
priority=RECOMMEND&\
car_type=1&\
alternatives=false" \
  -H "Authorization: KakaoAK ${REST_API_KEY}"
```

> 정확한 파라미터·응답 구조는 모빌리티 공식 레퍼런스(`/guide/navi-api/reference.html`)를 함께 확인할 것. WebFetch 단계에서 SPA 본문 미수집 — 추후 수동으로 보강 필요.

## 런트립 활용 매핑

| 기획안 기능 | 사용 방식 |
|---|---|
| **D-1 동선** (저강도·짧은 이동·조기 복귀) | `/v1/directions` — 숙소 → 식당 → 숙소 단순 경로 |
| **D-day** (대회장 ↔ 골인 직후 식당) | `/v1/directions` — 숙소 → 대회장 / 결승선 → 회복 식당 |
| **D+1** (회복·웰니스 코스) | `/v1/waypoints/directions` — 다중 경유지로 스파/사진/관광지 묶음 |
| **동반자 분리형 트랙** | 다중 목적지 = `/v1/destinations/directions` — 결승 직전 합류지점까지의 ETA 동기화 |
| **"가장 한산한 도시 추천"** | KTO #20 도시 결정 후, 그 도시까지의 출발지 다수 비교 시 `/v1/origins/directions` |
| **부산 광안리·해운대 코스 미래 시간 운행** | `/v1/future/directions` — 대회 D-1 토요일 18:00 출발 등 |

## 좌표계 / 주의사항

- 좌표는 **경도(x), 위도(y)** 순서가 일반적 (카카오 REST 계열 일관 — 공식 레퍼런스로 최종 확인)
- 결과의 거리/시간 단위, 폴리라인 인코딩 방식은 응답 본문 기준으로 확인
- 트래픽 한도 초과 시 429 → 가격 페이지를 통해 별도 협의
