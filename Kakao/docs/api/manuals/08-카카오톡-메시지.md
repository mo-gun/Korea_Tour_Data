# 08. 카카오톡 메시지 (KakaoTalk Message)

> 출처: <https://developers.kakao.com/docs/ko/kakaotalk-message/common>

서비스가 사용자 본인 또는 친구에게 카카오톡 메시지를 능동 발송하는 API. 사용자 액세스 토큰 + 동의항목 필수.

## 발송 모드

- **나에게 보내기** — 사용자 본인의 "나와의 채팅방"
- **친구에게 보내기** — 친구 목록 조회 API 결과 친구

## 카카오톡 공유와의 차이

| 항목 | 카카오톡 공유 | 카카오톡 메시지 |
|---|---|---|
| 발송 트리거 | 사용자가 직접 선택 | 서비스가 능동 호출 |
| 권한 | 불필요 | **필수** (앱 멤버 외 추가 검수) |
| REST API | 일부 | 전체 지원 |
| UI | SDK가 피커 제공 | 자체 구현 필요 |

## 필수 인증

- 사용자 **액세스 토큰**
- 동의항목:
  - **카카오톡 메시지 전송** (`talk_message`)
  - 친구에게 보내기 시 추가: **카카오 서비스 내 친구목록** (`friends`)

## 주요 엔드포인트

| 기능 | 메서드 | URL |
|---|---|---|
| 나에게 (기본 템플릿) | POST | `/v2/api/talk/memo/default/send` |
| 나에게 (사용자 정의) | POST | `/v2/api/talk/memo/send` |
| 친구에게 (기본 템플릿) | POST | `/v1/api/talk/friends/message/default/send` |
| 친구에게 (사용자 정의) | POST | `/v1/api/talk/friends/message/send` |

호스트: `https://kapi.kakao.com`

### 인증 헤더
```
Authorization: Bearer {USER_ACCESS_TOKEN}
Content-Type: application/x-www-form-urlencoded
```

## 메시지 템플릿 종류

- **기본 템플릿** — 사전 정의된 형식
- **사용자 정의 템플릿** — 템플릿 도구로 직접 구성 (파라미터 치환 가능)
- **스크랩 메시지** — 웹페이지 URL 기반

## 정책 / 제약

- 비공개 프로필 사용자에게는 발송 불가
- **쿼터 한도 상향 불가** (정책)
- 친구에게 보내기는 사용 권한 신청 필수

## 지원 플랫폼

REST API · JavaScript · Android · iOS · Flutter

## 런트립 활용 매핑 (단계적)

| 기획안 기능 | 사용 방식 | 단계 |
|---|---|---|
| **D-1 알림** ("내일 부산바다마라톤! 회복 식당은 ...") | 나에게 보내기 (사용자 정의 템플릿) | V2 |
| **결승 직전 동반자 호출** ("러너가 곧 도착! 합류 지점은 ...") | 친구에게 보내기 (선택 동행자) | V2 (권한 심사 후) |
| **"가볍게 뛸까요?" 추천 푸시** | 나에게 보내기 (사용자 동의 시) | V2 |
| **B2B 마케팅·재방문 유도** | 카카오톡 채널 + 알림톡 (별도 비즈니스 메시지)으로 일반적 운영 | V2.5+ |

## 도입 권장 시점

- **MVP**: 카카오톡 공유만 사용 (권한 불필요)
- **V2**: 비즈 앱 전환 후 카카오톡 메시지 사용 권한 신청 → 나에게 보내기부터 시작
- **V2.5**: 친구에게 보내기 권한 검수 통과 후 동반자 알림 사용
