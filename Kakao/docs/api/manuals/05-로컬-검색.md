# 05. 로컬 (Kakao Local) — 검색·지오코딩

> 출처: <https://developers.kakao.com/docs/ko/local/common> · <https://developers.kakao.com/docs/ko/local/dev-guide>

런트립 부가 기능 **"피니시라인 리프레시 정보"** (사우나·찜질방·짐보관·코인샤워 반경 검색) 와 **숙소·식당 추천** 의 핵심 검색 엔진. 한국관광공사 OpenAPI는 *관광 자산*만 다루므로, 일반 상권(편의점·약국·찜질방 등)은 카카오 로컬이 보완.

## 인증 헤더

```
Authorization: KakaoAK ${REST_API_KEY}
```

## 엔드포인트

| 기능 | 메서드 | URL |
|---|---|---|
| 주소 → 좌표 | GET | `https://dapi.kakao.com/v2/local/search/address.${FORMAT}` |
| 좌표 → 행정구역 | GET | `https://dapi.kakao.com/v2/local/geo/coord2regioncode.${FORMAT}` |
| 좌표 → 주소 | GET | `https://dapi.kakao.com/v2/local/geo/coord2address.${FORMAT}` |
| 좌표계 변환 | GET | `https://dapi.kakao.com/v2/local/geo/transcoord.${FORMAT}` |
| 키워드 장소검색 | GET | `https://dapi.kakao.com/v2/local/search/keyword.${FORMAT}` |
| 카테고리 장소검색 | GET | `https://dapi.kakao.com/v2/local/search/category.${FORMAT}` |

`${FORMAT}` 미지정 시 JSON 응답.

## 주요 요청 파라미터

### 키워드 장소검색
| 파라미터 | 필수 | 설명 |
|---|---|---|
| `query` | ✓ | 검색어 (예: "찜질방", "코인샤워") |
| `category_group_code` |  | 결과 카테고리 필터 (아래 표) |
| `x`, `y` |  | 중심 좌표 — 결승선 좌표 입력 |
| `radius` |  | 0 ~ 20000 (m) |
| `rect` |  | 사각 영역 검색 (`x1,y1,x2,y2`) |
| `page` |  | 1 ~ 45 |
| `size` |  | 결과 개수 |
| `sort` |  | `accuracy`(기본) / `distance` |

### 카테고리 장소검색
| 파라미터 | 필수 | 설명 |
|---|---|---|
| `category_group_code` | ✓ | 카테고리 그룹 코드 1개 |
| `x`, `y`, `radius` | (택1) | 원형 영역 |
| `rect` | (택1) | 사각 영역 |
| `page`, `size`, `sort` |  | 위와 동일 |

### 주소 → 좌표
| 파라미터 | 필수 | 설명 |
|---|---|---|
| `query` | ✓ | 주소 |
| `analyze_type` |  | `similar` / `exact` |
| `page`, `size` |  | 페이징 |

## 호출 예시

### 결승선 반경 2km 찜질방 (키워드 검색)
```bash
curl -G "https://dapi.kakao.com/v2/local/search/keyword.json" \
  --data-urlencode "query=찜질방" \
  --data-urlencode "x=129.158" \
  --data-urlencode "y=35.158" \
  --data-urlencode "radius=2000" \
  --data-urlencode "sort=distance" \
  -H "Authorization: KakaoAK ${REST_API_KEY}"
```

### 결승선 반경 1km 음식점 (카테고리 검색)
```bash
curl -G "https://dapi.kakao.com/v2/local/search/category.json" \
  --data-urlencode "category_group_code=FD6" \
  --data-urlencode "x=129.158" \
  --data-urlencode "y=35.158" \
  --data-urlencode "radius=1000" \
  --data-urlencode "sort=distance" \
  -H "Authorization: KakaoAK ${REST_API_KEY}"
```

### 주소 → 좌표
```bash
curl -G "https://dapi.kakao.com/v2/local/search/address.json" \
  --data-urlencode "query=부산 해운대구 구남로 12" \
  -H "Authorization: KakaoAK ${REST_API_KEY}"
```

## 주의사항

- **거리 단위**: 미터(m)
- **page 범위**: 1 ~ 45
- **radius 범위**: 0 ~ 20000 m
- **좌표계 지원**: WGS84 / WCONGNAMUL / CONGNAMUL / WTM / TM / KTM / UTM / BESSEL / WKTM / WUTM
- 카카오 좌표 기본은 `x = 경도(longitude)`, `y = 위도(latitude)` — 카카오맵 SDK의 `LatLng(위도, 경도)` 와 인자 순서 반대. 변환 필수.

## 런트립 활용 매핑

| 기획안 기능 | 사용 방식 |
|---|---|
| **피니시라인 리프레시** (사우나·찜질방·짐보관·코인샤워) | 키워드 검색 — `query=찜질방/사우나/코인샤워/짐보관`, 결승선 좌표 + radius=2000 |
| **D-day 골인 후 회복 식당** | 카테고리 검색 `FD6` (음식점), 결승선 좌표 반경 1km, 정렬 distance |
| **D+1 카페** | 카테고리 검색 `CE7` |
| **응급 약국·병원** (대회 안전망) | `PM9` / `HP8` 카테고리 |
| **부산 외 지역의 KTO 빈자리 보완** | 비파일럿 지역에서 KTO #21·#22로 부족한 일반 POI를 카카오 카테고리로 보강 |
| **주소 검색 → 좌표 → KTO `locationBasedList2` 호출** | 사용자 입력 주소를 좌표로 변환해 KTO 위치기반 API에 전달 |
| **카카오맵 클릭 좌표 → 행정구역** | `coord2regioncode` — 시·군 단위 KTO #20(혼잡도 예측)에 매핑 |

## 도메인/IP 등록

REST API Key는 카카오디벨로퍼스 → 앱 → 플랫폼 → Web 또는 IP 허용 목록에 등록. 서버에서만 호출.
