# 10. 카카오페이 (KakaoPay) — 온라인 결제

> 출처: <https://developers.kakaopay.com/docs/payment/online/common> · <https://developers.kakao.com/product/kakaoPay>

런트립의 **발전 방향(경량 제휴·예약·리워드)** 단계에서 사용. MVP에는 포함되지 않음. 카카오 본사가 아닌 **카카오페이 디벨로퍼스(별도 사이트)** 에서 가맹점 가입 후 키 발급.

문서 메뉴 기준으로 다음 카테고리 제공:

## 일반적 결제 흐름 (단건 결제 표준)

```
[1] 가맹점 서버 → 카카오페이 결제 준비 (ready) 요청
   ↓ tid + next_redirect_pc/mobile/app_url 응답
[2] 사용자 → 카카오페이 결제창에서 결제 진행
   ↓ approval_url / cancel_url / fail_url 콜백
[3] 가맹점 서버 → 결제 승인 (approve) 요청
   ↓ 결제 완료 응답
```

## 인증 / 사전 설정

1. 카카오페이 가맹점 가입
2. 카카오페이 디벨로퍼스에 앱 등록
3. **Secret Key** 발급 (서버 전용)
4. 테스트 / 운영 CID 부여
5. 호출 시 `Authorization: SECRET_KEY {SECRET_KEY}` 형식 (정확한 헤더 형식은 공식 문서 최종 확인)

## 가맹점 / CID

- **CID** — 가맹점 식별자. 결제 종류·테스트 환경별로 다름
- **테스트 CID**: 통상 `TC0ONETIME` 등 (공식 가이드에서 최신값 확인)
- 운영 CID는 가맹점 가입 후 카카오페이가 발급

## 런트립 활용 매핑 (모두 V2.5+)

| 발전 방향 항목 | 결제 모델 |
|---|---|
| **레이트체크아웃 패키지** (지역 숙소 제휴) | 단건 결제 |
| **짐보관 패키지** | 단건 결제 |
| **셔틀·참가권 대행** (별도 사업체 협업 단계) | 단건 결제 |
| **B2B SaaS 구독료** (지자체 활성화 KPI 대시보드) | 정기 결제 |
| **카카오페이 마일리지 + 디지털 관광주민증** | 부가 기능 + 외부 정책과 결합 |

## 도입 권장 시점

- **MVP**: 사용 X — 정보 제공형 운영
- **V2.5+**: 사용자/지자체 검증 후 *경량 제휴* 단계에서 단건 결제 도입
