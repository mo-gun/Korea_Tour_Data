# 06. 카카오내비 (Kakao Navi)

> 출처: <https://developers.kakao.com/docs/ko/kakaonavi/common>

App-to-App 방식으로 **카카오내비 앱을 실행**하거나, **자체 서비스에 내비게이션 SDK를 임베드**하는 두 가지 모드.

## 카카오 모빌리티 길찾기 API와의 차이

| 구분 | 카카오내비 (Kakao Navi) | 카카오 모빌리티 길찾기 API |
|---|---|---|
| 위치 | developers.kakao.com | developers.kakaomobility.com (별도 사이트) |
| 본질 | 카카오내비 *앱을 띄우는* SDK / 앱 임베드 SDK | REST API로 *서버가 직접* 경로 계산 |
| 결과 사용 위치 | 카카오내비 앱 또는 임베드 화면 | 서비스 자체 지도/UI |
| 런트립 활용 | D-day 대회장 길안내 트리거 (사용자에게 즉시 내비 띄우기) | 추천 일정의 ETA 계산, UI에 시간 표시 |

## 제공 기능

- **앱 실행 (App-to-App)** — `Kakao.Navi.start()` / `Kakao.Navi.share()`
- **내비게이션 SDK** — 자체 앱 내 임베드 (커스터마이즈)
- **내비게이션 SDK with UI** — 사전 제작된 UI 임베드
- **REST API 길찾기** — (모빌리티 길찾기와 별도로 카카오 측 가이드 존재)

## 지원 플랫폼

JavaScript · Android · iOS · Flutter · REST API

## 주요 메서드 (JavaScript SDK)

| 함수 | 용도 |
|---|---|
| `Kakao.Navi.start({ name, x, y, coordType })` | 카카오내비 앱 실행 + 길안내 |
| `Kakao.Navi.share({ name, x, y, coordType })` | 목적지 공유 |

## 런트립 활용 매핑

| 기획안 기능 | 사용 방식 |
|---|---|
| **D-day 대회장 출발 시 길안내** | 카드의 "내비로 출발" 버튼 → `Kakao.Navi.start()` 즉시 카카오내비 호출 |
| **결승 후 회복 식당까지 안내** | 골인 직후 푸시 또는 카드 버튼 → `Kakao.Navi.start()` |
| **동반자 트랙의 합류 지점 안내** | 동반자 폰에서 결승 직전 도달지로 자동 길안내 |
| **추천 일정의 외부 내비 위임** | 사용자 본인 내비 선호 존중 — 카카오맵 URL 스킴(`/link/to/`) 또는 카카오내비 SDK 선택 |

## 쿼터

월간/일간 쿼터 적용. 한도 상향은 디벨로퍼스 채널 협의.

## 도입 우선순위

- **MVP**: 카카오맵 SDK + 카카오 모빌리티 REST 조합으로 ETA·동선 표시 → "내비로 출발" 버튼 클릭 시 카카오내비 SDK 호출
- **V2**: 내비게이션 SDK 임베드 검토 (서비스 내부에서 끝까지 안내)
