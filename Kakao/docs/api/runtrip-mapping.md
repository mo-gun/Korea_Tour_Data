# 런트립 ↔ Kakao API 매핑

기획안(`/Users/geonmo/workspace/RunTrip/런트립_최종_기획안.md`)의 기능별로 어떤 카카오 API를 어디에 사용하는지 한 장으로 정리.

## 메인 기능 1 — 「전국 마라톤 통합 캘린더 + 검색」

> KTO `searchFestival2` + 자체 크롤링이 핵심. 카카오 API 비중은 낮음.

| 하위 기능 | 카카오 API |
|---|---|
| 사용자 로그인 | **카카오 로그인** → V2에서 **카카오싱크** |
| 대회 카드 카톡 공유 | **카카오톡 공유** (`feed` 템플릿) |

## 메인 기능 2 — 「대회 선택 → 카카오맵 기반 전·후 여행 동선 추천」

> 카카오 API가 가장 많이 결합되는 핵심 깔때기.

| 시점 | 사용 API |
|---|---|
| 코스 영역·POI·동선 시각화 | **카카오맵 JS SDK** (`Map`, `Marker`, `Polyline`, `Circle`, `MarkerClusterer`) |
| 숙소 ↔ 대회장 ↔ 관광지 실제 이동시간 | **카카오 모빌리티 길찾기 API** (`/v1/directions`, `/v1/waypoints/directions`) |
| D+1 다중 경유지 회복 코스 | **카카오 모빌리티** `/v1/waypoints/directions` |
| 동반자 분리형 트랙 ETA 동기화 | **카카오 모빌리티** `/v1/destinations/directions` |
| KTO POI 외 일반 상권(식당·카페·숙박) 보강 | **로컬 카테고리 검색** (`FD6`, `CE7`, `AD5`) |
| 사용자 입력 주소 → 좌표 → KTO 위치기반 | **로컬 주소→좌표** (`/v2/local/search/address`) |
| 좌표 → 시·군 매핑 (KTO #20 혼잡도 예측 호출) | **로컬 좌표→행정구역** (`/v2/local/geo/coord2regioncode`) |
| D-day "내비로 출발" 버튼 | **카카오내비** (`Kakao.Navi.start()`) 또는 카카오맵 URL 스킴 `/link/to/` |
| 추천 일정 카톡 공유 | **카카오톡 공유** (`list` 템플릿) |

## 부가 기능 — 「피니시라인 리프레시」

| 하위 기능 | 카카오 API |
|---|---|
| 결승선 반경 2km 사우나·찜질방·짐보관 | **로컬 키워드 검색** `query=찜질방/사우나/코인샤워/짐보관`, `radius=2000` |
| 결승선 반경 1km 음식점·카페 | **로컬 카테고리 검색** `FD6`, `CE7` |
| 응급 약국·병원 | **로컬 카테고리 검색** `PM9`, `HP8` |
| 시각화 | **카카오맵** `Marker` + `MarkerClusterer` |

## 부가 기능 — 「가볍게 뛸까요?」

> KTO #17 (두루누비) + #20 (혼잡도) 결합. 카카오는 시각화·길안내 보조.

| 하위 기능 | 카카오 API |
|---|---|
| 코스 시각화 | **카카오맵** `Polyline` |
| 좌표 → 행정구역 매핑 | **로컬** `coord2regioncode` |
| 출발 안내 | **카카오내비** 또는 카카오맵 `/link/to/` |

## 발전 방향 매핑

| 발전 방향 항목 (5단계 로드맵) | 카카오 API |
|---|---|
| (1) 부산 → 강원·제주 지역 템플릿 복제 | 동일 (지도/모빌리티/로컬) |
| (2) 사용자 GPS 누적 → 표준 코스 데이터셋 | (별도 — 카카오 SDK는 시각화만) |
| (3) 경량 제휴 (레이트체크아웃·짐보관·샤워) | **카카오페이** (단건) + **카카오톡 채널** |
| (4) B2B SaaS — 지자체 활성화 KPI 대시보드 | **카카오페이** (정기) + **카카오톡 채널** (CRM) |
| (5) 다국어 인바운드 + 디지털 관광주민증·카카오페이 마일리지 | **카카오페이** + **카카오싱크** + 카카오 i 다국어 |

## 키 운영 매트릭스

| 키 | 사용 위치 | 사용 API |
|---|---|---|
| **JavaScript Key** | 프론트(Next.js) | 카카오맵, 카카오 로그인(JS), 카카오톡 공유(JS), 카카오내비(JS) |
| **REST API Key (kakao.com)** | Spring Boot 서버 | 카카오 로그인 토큰 교환, 로컬, 카카오톡 메시지, 카카오톡 채널 |
| **REST API Key (kakaomobility.com)** | Spring Boot 서버 | 모빌리티 길찾기 — **별도 발급** |
| **Secret Key (카카오페이)** | Spring Boot 서버 | 카카오페이 결제 — V2.5+ |
| **Admin Key** | (사용 안 함) | — |

## 한국관광공사 OpenAPI와의 결합 지점 (기획안 3.2 (1) 발췌)

기획안의 데이터 흐름도와 1:1 매핑:

```
[기능 2] 대회 선택 → 전·후 여행 동선 추천
   ├── KTO #1 detailIntro2        (대회 상세)
   ├── KTO #1 locationBasedList2  (코스 영역 내 POI)
   │       ↑ 좌표 입력값은 사용자 검색어 → 카카오 로컬 주소→좌표 변환 결과
   ├── KTO #21 + #22              (중심·연관 관광지 체이닝)
   ├── KTO #25                    (D+1 웰니스)
   ├── KTO #13                    (UI 사진)
   ├── KTO #11 / #23              (옵션: 무장애·반려동물)
   └── 카카오맵 + 카카오 모빌리티  (실제 이동시간·동선 시각화)  ← 본 폴더
                                    + 카카오 로컬 (일반 상권 보강)
                                    + 카카오내비 (실주행 트리거)

[부가] 피니시라인 리프레시
   └── 카카오 로컬 (찜질방/짐보관/코인샤워 키워드·카테고리 검색)
```

> *KTO OpenAPI는 추천 대상을 제공하고, 카카오 API는 그것을 **실행 가능한 동선·예약·공유** 로 바꾼다.*
